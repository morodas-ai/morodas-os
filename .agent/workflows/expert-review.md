# Expert Review（5段階・完全版）

> 詳細ルールは [expert-rules.md](./../experts/expert-rules.md) を参照

## 使い方

- `/expert-review` → 全段階フル実行
- 「第2段階まで」等で部分実行も可
- **軽微変更**（typo/コメント/README/CSS微調整）→ **免除**（スキャン不要）
- **緊急ホットフィックス**（本番障害）→ **まず直す。事後に第1段階のみ**

## 第4段階：神の目

**5つの問い:**

| 問 | 内容 |
|----|------|
| 問1 | **全員が見ていない方向**は何か？ |
| 問2 | **専門家同士の矛盾**はないか？（優先: セキュリティ > データ > UX > 開発効率） |
| 問3 | **1年後の自分**はこの判断をどう評価するか？ |
| 問4 | **そもそもこの作業自体が不要**ではないか？ |
| 問5 | このプロセスの**改善点を1つだけ挙げよ**（「なし」は禁止） |

## 出力記号

| 記号 | 意味 | 備考 |
|:---:|------|------|
| ✅ | チェック済み、問題なし | |
| ➖ | 非該当 | **理由必須** |
| 🔴 | Critical（データ損失/セキュリティ/本番障害） | **即停止。オジキに報告** |
| 🟡 | Warning（品質低下/技術的負債） | 同タスク内で修正 |
| 🔵 | Info（改善の余地あり） | ログ記録のみ |

---

## 原則

1. **「選ぶ」のではなく「検出する」** — トリアージ禁止
2. **段階は深さの違い** — 範囲の絞り込みではない
3. **1つのAIが全役を演じる限界を認識** — 広さは担保、深さは限界あり
4. **収穫逓減を受け入れる** — 実戦で磨く。机上で磨きすぎない
